# OCR_Checker_development

---

## I. Структура проекта

__Проблема__ - учителю нужно быстро и честно проверять ДЗ по математике или информатике, включая рукописные решения.

__Цель платформы__ - автоматическая проверка "по условию и по методу": верность ответа и корректность решения.

__Ключевые функции__:
1. Загрузка условий, эталонных решений и ответов;
2. OCR рукописных и печатных работ;
3. Проверка ответов (математика: символика; информатика: код + тесты);
4. Анализ шага решения (детектор пропусков и ошибок);
5. Отчет с подсветкой расхождений.

__Технологический стек__:
1. Backend: FastAPI + Celery + PostgreSQL;
2. ML: OCR (PaddleOCR or Tesseract), формульный парсер (Latex), sympy;
3. Code judge: изолированный sandbox (Docker, seccomp), unit-тесты;
4. Frontend: Next.js or React;
5. Хранение: MinlO/S3.

__Безопасность и антиплагиат__:
1. Детектор копипасты по AST;
2. Проверка последовательности шагов решения;
3. Изолированное исполнение кода.

__Поток данных__: 
- Задача
- Эталон
- Работа ученика (pdf/jpg/py/ipynb)
- OCR/Parser
- Проверка
- Отчет

__Метрики качества__ - TPR or FPR OCR, точность парсинга формул, % автоматических вердиктов без ручной доразметки, время проверки.

__Экономика модели__:
1. Экономия времени преподавателя;
2. Снижение списывания;
3. Данные для методистов.

__Риски__:
1. Качество OCR;
2. Шумные фото;
3. Безопасность sandbox;
4. Подготовка тестов.

---
## II. План работ

__Этап 1 (Подготовка)__:
1. Уточнение или определение формата входных данных, ТЗ на отчет;
2. Собрать 300-500 примеров рукописных решений различных учеников.

__Этап 2 (Архитектурный скелет)__:
1. FastAPI + PostgreSQL + MinlO + Celery + Redis;
2. Docker compose, аутендификация и загрузка файлов.

__Этап 3 (Модуль OCR и пасринг)__:
1. Интеграция PaddleOCR or Tesseract, пост-процессинг формул и нормализация;
2. Базовая модель классификации типа задачи (математика или информатика).

__Этап 4 (Проверка решений)__:
1. Математика: SymPy для упрощения и сопоставления выражений, эвристики проверки шага (например: упорядочивание и эквивалентность);
2. Информатика: генераторо тесткейсов + запуск в песочнице, AST-анализ на запрещенные конструкции и детекция копипасты.

__Этап 5 (Качество и безопасность)__:
1. Метрики OCR, нагрузочное тестирование6 WAF/Rate-limit, S3-политики;
2. Документация и демо-презентация.

---

## III. Архитектура

- __Gateway / Frontend__: Next.js, Aut;
- __API__: FastAPI, отдает задания, прием решений, выдача отчетов;
- __Storage__: MinlO/S3 для файлов, PostgreSQL для метаданных, Redis для очередей;
- __Workers__: Celery - OCR, парсинг, оценка; Judge - безопасный запуск кода в Docker;
- __ML__: OCR + формулы (PaddleOCR + LaTeX parser), SynPy checker, Code AST checker;
- __Monitoring__: Prometheus + Grafana, Sentry.
